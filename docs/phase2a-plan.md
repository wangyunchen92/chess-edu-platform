## 1. Phase 2a 范围定义

Phase 2 拆分为两个子阶段：

| 子阶段 | 范围 | 优先级 |
|--------|------|--------|
| **Phase 2a（本文档）** | AI角色扩展、Level 2-3 课程、弱点诊断、自适应难度优化 | 优先交付 |
| Phase 2b（后续规划） | LLM角色对话、大师会员+家庭方案、家长面板、视频课程、金币商城、复盘分享 | Phase 2a 完成后启动 |

### Phase 2a 四大功能

| # | 功能 | 描述 |
|---|------|------|
| F1 | AI角色扩展 | 初级段3角色（冬冬、狸花花、铁墩墩）+ 中级段3角色（银鬃、咕噜、云朵师父），共6个新角色 |
| F2 | 课程内容 | Level 2（基础战术，约15课）+ Level 3（中级战略，约15课） |
| F3 | 弱点诊断 | 分析用户做题/对弈数据，识别薄弱环节（如开局、战术计算、残局），推荐专项训练 |
| F4 | 自适应难度 | 优化AI对弈难度算法：动态调整引擎参数，维持40%-60%目标胜率区间 |

## 3. 依赖关系分析

```
                    ┌──────────────┐
                    │ F1 AI角色扩展 │
                    └──────┬───────┘
                           │
            ┌──────────────┼──────────────┐
            │              │              │
            ▼              ▼              ▼
    ┌──────────────┐ ┌──────────┐ ┌──────────────┐
    │ F2 课程内容   │ │ F4 自适应 │ │ F3 弱点诊断   │
    │ (Level 2-3)  │ │ 难度优化  │ │              │
    └──────────────┘ └──────────┘ └──────────────┘
            │                            │
            └──────────┬─────────────────┘
                       ▼
              ┌────────────────┐
              │ 弱点诊断推荐    │
              │ 专项训练/课程   │
              └────────────────┘
```

### 依赖矩阵

| 功能 | 前置依赖 | 被依赖 |
|------|---------|--------|
| F1 AI角色扩展 | 无（可立即启动） | F4 需要新角色来验证自适应算法 |
| F2 课程内容 | 无（可立即启动） | F3 弱点诊断推荐需要有课程可推荐 |
| F3 弱点诊断 | 部分依赖 F1（需要更丰富的对弈数据维度）和 F2（推荐内容） | 无 |
| F4 自适应难度 | 部分依赖 F1（需在不同水平角色上验证） | F3 可参考自适应系统的数据 |

### 关键结论

- **F1 和 F2 无前置依赖**，可最先启动且互不阻塞
- **F4 自适应难度**可与 F1 并行开发，但需要 F1 完成后才能做集成测试
- **F3 弱点诊断**是最晚启动的功能，需要 F1 提供更多角色数据、F2 提供推荐课程

## 7. 优先级排序与风险

### 7.1 优先级排序

| 优先级 | 功能 | 理由 |
|--------|------|------|
| **P0** | F1 AI角色扩展 | 核心内容，6个新角色直接扩展产品可玩内容量，是用户升级路径的关键 |
| **P0** | F2 Level 2 课程 | 初级段用户晋升到战士段必须完成 Level 1 + Level 2 课程，是升段硬性条件 |
| **P1** | F4 自适应难度 | 显著提升对弈体验，但即使没有，固定 Rating 角色仍可正常对弈 |
| **P1** | F2 Level 3 课程 | 中级段晋升需要，但优先级低于 Level 2（用户还没到那个阶段） |
| **P1** | F3 弱点诊断 | 增值功能，提升留存和训练针对性，但不影响核心流程 |

### 7.2 可降级方案

如果工期紧张，以下内容可降级或延后：

| 可降级项 | 降级方案 |
|---------|---------|
| Level 3 课程 | 先出大纲和5课，剩余延后 |
| 弱点诊断推荐引擎 | 先做弱点展示，推荐用硬编码规则代替智能匹配 |
| 角色解锁剧情动画 | 先用纯文字弹窗，动画效果延后 |
| 难度指示器UI | 后台生效即可，前端展示延后 |
| 陷阱走法生成器 | 先用常规引擎参数模拟，不做专门的陷阱逻辑 |

### 7.3 主要风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 内容制作瓶颈（30课+6角色对话） | 阻塞 Sprint 2a-1 后端/前端开发 | content-agent 最先启动，Sprint 2a-0 全力制作内容 |
| 自适应难度算法调参困难 | 用户体验不佳（太难或太简单） | 先上线简单版（近5局胜率调整），后续迭代 |
| 弱点诊断数据不足 | 新用户数据少，诊断不准 | 设最低数据门槛（至少10局对弈+30道谜题后才出诊断） |
| 引擎棋风差异不明显 | 6个角色"下棋感觉一样" | chess-engine-agent 重点攻关棋风差异化，QA 对比验证 |
| 角色美术资源缺失 | UI 不完整 | 先用 AI 生成占位图，Phase 2b 换专业美术 |

## 8. 验收标准

### F1 AI角色扩展
- [ ] 角色大厅展示12个角色（3入门+3初级+3中级），初级和中级角色显示锁定状态
- [ ] 每个角色有独立的性格化对话（至少12场景 x 5条）
- [ ] 初级段角色在用户通过「草原守护者之战」晋级挑战后解锁
- [ ] 中级段角色在用户通过「森林之心」晋级挑战后解锁
- [ ] 每个角色的对弈感受有明显差异（棋风不同）
- [ ] 好感度系统正常累积

### F2 课程内容
- [ ] Level 2 课程（15课）在课程列表可见，需完成 Level 1 后解锁
- [ ] Level 3 课程（15课）在课程列表可见，需完成 Level 2 后解锁
- [ ] 每课包含讲解 + 互动棋盘 + 练习题，练习可提交并计分
- [ ] 课程进度正确记录，完成后获得对应经验值

### F3 弱点诊断
- [ ] 用户完成足够数据量后（10局+30题），可查看弱点画像
- [ ] 弱点画像以雷达图/柱状图展示各维度能力值
- [ ] 薄弱环节自动推荐对应谜题包、课程、训练计划
- [ ] Dashboard 首页展示弱点摘要

### F4 自适应难度
- [ ] 连续胜利时 AI 明显变强（体感可感知）
- [ ] 连续失败时 AI 明显变弱（体感可感知）
- [ ] 胜率趋向40%-60%区间稳定
- [ ] 对弈结算页可查看当局难度参数（可选）
