## API 接口细节说明

### 进度标记接口要求

`POST /api/v1/learn/lessons/{id}/progress` 必须包含三个字段：

```json
{
  "completed_steps": <步数>,
  "status": "completed",
  "progress_pct": 100
}
```

注意：若缺少 `progress_pct` 字段，接口返回 422 Validation Error。前端调用时需确保三字段齐全。

### exercises 存储位置

练习题存储在 `GET /api/v1/learn/lessons/{id}` 响应的 `data.content_data.exercises` 数组中，而**不是**通过独立的 `/exercises` 端点获取。

## 问题清单

### 问题1：`/lessons/{id}/exercises` 端点始终返回空数组（低优先级）

- **现象**：`GET /api/v1/learn/lessons/l0_01/exercises`、`l1_01/exercises`、`l1_10/exercises` 均返回 `{"code":0,"data":[]}`
- **实际情况**：练习题数据已存在，在 `GET /api/v1/learn/lessons/{id}` 的 `content_data.exercises` 字段中可以正常读取
  - l0_01：3道题（例：国际象棋的棋盘是几乘几的？）
  - l1_01：3道题，l1_06~l1_15：各1道题
- **影响**：如果前端有代码单独调用 `/exercises` 端点展示题目，将无法获取到任何题目
- **建议**：统一从 `content_data.exercises` 读取，或修复 `/exercises` 端点的数据查询逻辑

### 问题2：骨架课内容较少（已知设计，仅记录）

- **现象**：l1_06~l1_15 每课仅2步1题，内容极简
- **影响**：学习体验较短，但功能上完全可用，完成标记正常
