# 棋育宣传方案 v1 · 落地页 + 首发 5 条视频

- 状态：已评审（设计阶段）
- 日期：2026-04-24
- 来源：商业模式方案推广策略第一阶段落地执行（`教育教学/商业模式-线上平台+线下培训班联动方案.md` 第 104-118 行）

## 1. 背景与目标

### 现状
- 棋育平台 Phase 2 功能已上线（对弈/谜题/冒险/学习/训练/弱点诊断/成长体系齐全）
- 商业模式第一阶段目标：**3 个月内获取 100 个线上用户 + 20 个新线下体验学员**
- 线上→线下漏斗需要"曝光 → 注册 → 激活 → 线下转化"的冷启动内容和工具

### 目标
宣传第一波物料 MVP，双轨产出：
1. **A 路径 · 视频内容**：5 条短视频脚本（成片可直接用现有视频流水线生产）
2. **B 路径 · 落地页**：`/landing` 极简引流页（配合海报二维码、视频评论区挂链）

### 非目标
- 不做微信登录（列入 P2 backlog）
- 不做游客模式（需要后端"临时账户"机制，1-2 天额外工期）
- 不做 7 天会员赠送的支付/会员扩展（P2）
- 本轮不拍真人出镜视频（纯录屏 + AI 配音 + 角色画面）

## 3. A 路径 · 视频脚本设计

### 3.1 5 条视频主题（从漏斗上到下：吸引 → 信任 → 转化）

| # | 类型 | 核心信息 | 家长钩子 |
|---|---|---|---|
| 1 | **痛点** | 家长不会下棋也没时间陪 → AI 老师 24h 陪练 | 省心 |
| 2 | **产品 IP** | 豆丁老师陪孩子对弈 · 萌角色让孩子自己爱上 | 孩子不抗拒 |
| 3 | **学习数据** | 家长每周收到 AI 学习报告 · 看数据不听口头反馈 | 看得见效果 |
| 4 | **情感瞬间** | 孩子考过「草原小考」拿毕业徽章 🌿 · 升级成就感 | 孩子有动力 |
| 5 | **AI 因材施教** | 弱点诊断雷达图 → 自动生成个性化训练计划 | **1 对 1 名师级能力**的差异化卖点 |

### 3.2 每条视频结构（30-45s）

```
[0-3s]    钩子镜头（家长痛点场景或孩子专注画面）+ 大字标题
[3-25s]   产品录屏演示（核心功能 3-5 个画面）
[25-35s]  价值主张（字幕 + 配音总结）
[35-45s]  CTA（扫码进入棋育 / 附近培训班体验）+ 邀请码角标
```

### 3.3 素材来源

| 素材 | 来源 | 备注 |
|---|---|---|
| 产品录屏 | Playwright 自动截图脚本 | 复用现有 `skill_video_production.md` 流水线 |
| AI 配音 | ChatTTS seed1111 | 温柔女声，儿童教育场景合适 |
| 开场/结尾画面 | Playwright setContent HTML | 大字体卡片 + 品牌色 |
| BGM | 开源/版权免费（如 Pixabay） | 童趣 + 温馨风 |
| 字幕 | drawtext 烧录（避开 SRT 踩坑）| 白字黑描边 fontsize=28 |

### 3.4 5 条成品脚本 → Task 3 完整产出

每条含：
- 标题（抖音/视频号/小红书各一版）
- 分镜脚本（时间轴 + 画面 + 配音 + 字幕）
- Playwright 截图清单（需要哪些页面）
- 发布文案（含话题标签）

## 4. B 路径 · 落地页设计

### 4.1 路由与层级

- **路由**：`/landing`
- **无需登录**：外部访客可直接访问
- **布局**：手机端优先（海报扫码主场景），PC 兼容

### 4.2 页面结构（Minimum）

```
┌─────────────────────────────┐
│                              │
│        🏰 棋境大陆            │
│                              │
│   让孩子在家，学会国际象棋     │
│   AI 陪练 · 每周数据 · 游戏化  │
│                              │
│   ┌─────────────────────┐    │
│   │   开始试玩 →        │    │
│   └─────────────────────┘    │
│                              │
│   已有 N 位小棋手加入         │
│                              │
│   备案号 · 联系方式           │
└─────────────────────────────┘
```

- LOGO + 大 slogan
- 1 个主 CTA 「开始试玩 →」
- 社会认同（用户数，可 mock 或真实计数）
- 底部 copyright / 备案号

### 4.3 CTA 行为

- 点击 「开始试玩」→ `navigate('/register?code=<邀请码>')`
- URL 里可选 `?code=XXX` 参数 → 登陆页自动预填邀请码字段
- 也可以走 `/register?ref=landing` 追踪来源

### 4.4 Good 档升级 · 新用户试玩引导

注册成功 → 登录状态 → 首次进入主站（`/`）时：
- 检测 `user.created_at` 和当前时间差 < 5 分钟 → 视为首次登录
- 自动弹一个「欢迎模态」：
  - 动画 + 豆丁老师形象
  - 文案："嗨！我是豆丁老师，现在跟我下第一局吧！"
  - 按钮「好！来一局」→ `navigate('/play/game/<douding_id>')` 自动开局
  - 按钮「先看看」→ 关闭模态，正常到首页

**实现位置**：`frontend/src/pages/dashboard/DashboardPage.tsx` 或 `App.tsx` 全局 hook

### 4.5 二维码生成

- 新增脚本 `scripts/gen-landing-qr.js`（Node）
- 输入：落地页 URL + 邀请码 → 输出：带品牌装饰的二维码 PNG（可直接印海报）
- 用 `qrcode` npm 包生成基础二维码，用 `sharp` 或 canvas 合成品牌装饰

## 6. 视频生产流程（参考，不属本 spec 的 MVP）

走 `memory/shared/skill_video_production.md` 的标准流水线：
1. 拿到 Task 3 的 5 条成品脚本
2. Playwright 截取所需页面 + 生成开场/结尾 HTML 卡片
3. ChatTTS 分段生成配音 wav
4. 每段 drawtext 烧录字幕 → 单段 mp4
5. concat 拼接 + 混入 BGM（15% 音量）
6. 输出 3 个版本（1080x1920 竖版抖音/视频号 + 1080x1350 竖版小红书 + 1920x1080 横版备份）

## 8. 风险与边界

- 落地页"首登录引导"的首次判定用 `user.created_at < 5min`，有小概率已注册老用户清 cookie 重登触发——可接受（引导无害）
- 视频脚本只是 outline，真拍还要拿到：
  - 演示账号（非 admin/student，用专门的演示账号）
  - 演示数据（漂亮的对弈、有代表性的报告）
  - BGM 版权
- 海报邀请码需要运营侧专门创建一批追踪不同渠道来源的邀请码（抖音 / 视频号 / 小红书 / 线下地推各一）
