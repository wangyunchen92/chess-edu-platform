# 复盘沉浸式页面设计

> 日期：2026-04-02 | 状态：已确认

## 需求

复盘页面改为沉浸式全屏体验，隐藏侧边栏和顶部导航，让孩子专注于复盘分析。

## 设计方案

### 路由变更

将 `/play/review/:id` 路由从 `<AppLayout>` 内部移到外层，使其成为独立全屏页面（类似登录页）。

**App.tsx 变更**：
```
// 从 AppLayout 内移出
<Route path="/play/review/:id" element={<ReviewPage />} />
```

### 页面结构

#### 顶栏（固定）
- 左：返回按钮（← 返回）
- 中：对局信息（你 vs 角色名 · 结果）
- 右：分析按钮 + 深度选择器（未分析时）/ 统计摘要（已分析时）

#### 桌面端（lg 及以上）
```
┌─────────────────────────────────────────────────┐
│  ← 返回   你 vs 豆丁 · 1-0    [深度▾] [分析对局] │  顶栏
├──────────────────────┬──────────────────────────┤
│                      │  统计卡片（分析后显示）    │
│     评估条            │                          │
│     ┌──────────┐     │  走法列表                 │
│     │          │     │  1. e4  e5               │
│     │  棋 盘   │     │  2. Nf3 Nc6              │
│     │  (60%)   │     │  3. Bb5 ...              │
│     │          │     │                          │
│     └──────────┘     │  局面小贴士               │
│     ◀ ◁ ▷ ▶ ⏯       │  "白棋稍微领先..."        │
│     播放控制栏        │                          │
├──────────────────────┴──────────────────────────┤
```

#### 移动端（< lg）
```
┌───────────────────────┐
│ ← 返回  你 vs 豆丁    │ 顶栏
├───────────────────────┤
│       评估条           │
│   ┌─────────────┐     │
│   │             │     │
│   │   棋 盘     │     │
│   │  (全宽)     │     │
│   │             │     │
│   └─────────────┘     │
│   ◀ ◁  ▷ ▶  ⏯        │ 播放控制
│   [深度▾] [分析对局]   │ 分析按钮
├───────────────────────┤
│   走法列表（可折叠）    │
│   局面小贴士           │
└───────────────────────┘
```

### 样式规范

- 背景：`#0b1120`（深色，与登录页一致）
- 顶栏：半透明深色 `rgba(11,17,32,0.9)` + backdrop-blur
- 卡片/面板：`rgba(30,41,59,0.6)` + 半透明边框
- 页面填满视口：`min-h-screen`
- 棋盘尽量大：桌面端约 480-560px，移动端 `100vw - 32px`

### 不涉及的变更

- 不改后端 API
- 不改数据结构
- 不改分析逻辑（ReviewAnalyzer / EngineManager）
- 不改 Chessboard 组件本身

### 影响范围

| 文件 | 变更 |
|------|------|
| `App.tsx` | ReviewPage 路由移到 AppLayout 外 |
| `ReviewPage.tsx` | 重写布局为沉浸式全屏，响应式适配 |

### 验收标准

1. 进入复盘页后无侧边栏、无顶部导航
2. 返回按钮可正常返回
3. 桌面端：左棋盘右面板布局
4. 移动端：上下堆叠，棋盘全宽
5. 分析功能正常工作（按钮、进度、标注、小贴士）
6. 深色背景沉浸式体验
